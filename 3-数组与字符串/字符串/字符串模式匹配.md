## 介绍——字符串模式匹配问题定义
- 在**目标串**中寻找**模式串**出现的**首位置**
- 给定两个字符串 `S` 和 `P` ，目标串 `S` 有 `n` 个字符，模式串 `P` 有 `m` 个字符，`m ≤ n` 。判断 `P` 是否为 `S` 的子串，如果是则返回 `P` 的第一个字符在 `S` 中的首位置；如果不是则返回 `-1`
- 参考题目：[LeetCode 28 找出字符串中第一个匹配项的下标](https://leetcode.cn/problems/find-the-index-of-the-first-occurrence-in-a-string/description/)

## 朴素模式匹配算法（Brute Force算法，暴力算法）
- 思路：每次失配后：
  - 模式串右移 1 位
  - 指针 i 回退到 i-j+1
  - 指针 j 回退到 0
- 参考代码：
    ```cpp {.line-numbers}
    int StringMatching(char *s, char *p){ // 返回 p 在 s 中的位置
        int n = strlen(s), m = strlen(p); // n为 s 的长度, m 为 p 的长度
        int i = 0; // 通过 i 扫描 s
        while(i <= n - m){ // 看 p 是否在 s 的第 i 个位置, 从 s[i] 开始与 p 逐字符比对
            int j = 0; // 通过 j 扫描 p
            while(j < m && s[i] == p[j]){
                i++; j++;
            }
            if(j == m) return i - m; // 匹配成功
            i = i - j + 1; // 本趟匹配失败，指针 i 回退到原位置 +1
        }
        return - 1; // p 不在 s 中
    }
    ```
- 时间复杂度：
  - **关键运算**：字符比较
  - 最坏时间复杂度 `O(nm)`
- 总结：
  - 优点：思路简单、实现简单
  - 缺点：最坏情况下时间复杂度高
  - 规律：
    - 字符集越大，单次比对成功的概率越小，时间越接近线性
    - 字符集越小，单次比对成功的概率越高，越接近最坏情况，算法时间性能越差

## 快速模式匹配KMP算法
- 优化思考：**利用模式串中子串的重复性，加速匹配过程**
  | 朴素模式匹配存在的问题（每次匹配失败后）| 可能的优化途径 |
  | :-: | :-: |
  | 模式串 P 仅右移 1 位 | 模式串 P 能否多移几位? |
  | 目标串 S 的匹配指针 i 回退 | 指针 i 可否不回退? |
  | 模式串 P 的匹配指针 j 回退至0 | 指针 j 可否不回退至 0 ? |
  - 注意：
    - 要选择右移位数较少的方案，避免漏掉可能的成功匹配
    - 因此可以找失配位置前的子串的**最长相等前后缀**
- 基本原理：
  - 设目标 $S = s_0s_1...s_{n-1}$ ，模式串 $P = p_0p_1...p_{m-1}$
  - 假设当前正进行某次匹配:
    - 有 $s_{i-j}...s_{i-1} = p_0...p_{j-1}$,但 $si \neq pj$,即在 $pj$ 的位置失配
    - 示例图：

    | $S$ | ... | $s_{i-j}$ | ... | ... | ... | ==$s_{i-k}$== | ==...== | ==$s_{i-1}$== | $s_i$ | ... | $s_{n-1}$ |
    | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: |
    | **关系** | 空 | $=$ | $=$ | $=$ | $=$ | $=$ | $=$ | $=$ | $\neq$ | 不匹配了 |
    | **$P$** | 空  | ==**$p_0$**== | ==**...**== | ==**$p_{k-1}$**== | **...** | ==**$p_{j-k}$**== | ==**...**== | ==**$p_{j-1}$**== | **$p_j$** | **...** |  |
    - 在失配位置前对应的子串 $p_0...p_{j-1}$ 中，找最长相等的前后缀，设其长度为 $k$ , 则有：$s_{i-k}...s_{i-1} = p_{j-k}...p_{j-1} = p_0...p_{k-1}$ ( **即画高亮的部分** )
  - 下次匹配时：
    - 右移 $P$ 使 $p_0...p_{k-1}$ 与 $s_{i-k}...s_{i-1}$对齐，使 $s_i$ 和 $p_k$ 继续比对
    - 目标串匹配指针 `i` 不动，模式串匹配 `j` 指针只需回退到 `k` ，即模式串右移 `j-k` 位 ( $k$ 标识了模式串 $P$ 在位置 `j` 失配后，下次匹配的位置 `k` 是 $p_0...p_{j-1}$ 的最长相等前后缀的长度)
    - 示例图：
    
    | $S$ | ... | $s_{i-j}$ | ... | ... | ... | ==$s_{i-k}$== | ==...== | ==$s_{i-1}$== | $s_i$ | ... | $s_{n-1}$ |
    | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: | :-: |
    | **关系** | | | | | | $=$ | $=$ | $=$ | $?$ |  |
    | **$P$** |  |  | |  |  | ==**$p_0$**== | ==**...**== | ==**$p_{k-1}$**== | **$p_k$** | **...** |  |
- 在 $p_j$ 处匹配失败后的处理：
  - $P$ 的下一个匹配位置 `k`（$P$ 中的哪个字符和 $S$ 中的失配字符重新匹配）可看做关于 `j` 的函数，即 $k = next(j) = p_0 …p_{j-1} 的最长相等前后缀的长度$
    - 匹配过程中若在 $p_j$ 处失配, 则 $P$ 下次匹配的位置即$next(j)$
    - 模式串右移 $j-next(j)$ 位
  - 可以通过预处理，把所有的 $next(j)$ 算出来，即有：$next$ 函数（失败函数、前缀函数）
   $$
    next(j) = 
    \begin{cases}
    -1, & j = 0 \\
    \max\left\{ k \mid p_0 \dots p_{k-1} = p_{j-k} \dots p_{j-1} \text{ 且 } 0 < k < j \right\}, & \text{存在}k \\
    0, & \text{其他}
    \end{cases}
   $$
    - $next$ 函数仅与模式串 $P$ 有关，而与目标串 $S$ 无关
    - 可以预先把 $P$ 中所有位置的 $next(j)$ 算出来，存入数组 `next[j]`
    - 匹配过程中若在 $P_j$ 失配，则直接令 `j = next[j]` 作为 $P$ 下次匹配的位置
- 示例代码：
  ```cpp {.line-numbers}
    const int maxm = 1e4 + 10;
    int KMP(char *S, char *P, int n, int m){
        // 目标串S长度为n，模式串P长度为m ，返回P在S中的位置
        int next[maxn], i = 0, j = 0; // i, j分别为S和P的匹配指针
        buildNext(P, next, m); // 计算 next 函数
        while(j < m && i < n){ // 关键运算元素比较  while 循环迭代次数不仅取决于 i ，还取决于 j
            if(j == -1 || S[i] == P[j]) i++, j++; // (使j增加)
            else j = next[j]; //确定 P 下次匹配位置(使j减小)
        }
        return (j == m) ? i - m : - 1; // 判断匹配成功了还是失败了
    }
  ```
  - 时间复杂度分析：`O(n+m)`
    - 扫描过程中 `i` 最多从 `0 至 n` , `i` 在循环内永远不减小, 最多增加 `n` 次, 每次加 `1` , 而 `j++` 与 `i++ `同步, 故 `j` 也最多增加 `n` 次, 每次加 `1`
    - `j` 最小是从 `-1` 开始增加, 即 `j++` 的最小起点是 `-1` , 迭代过程中永远不会小于 `-1 ` , 故 **`j` 减小（ `j =next[j]` ）的次数不会超过 `j` 增加的次数**, 即 `n` 次
    - 故关键运算不会超过 `2n` 次, 即 `O(n)`
- $next(j)$ 的计算的实现
  - 暴力方法:
    - 在 $p_0p_1...p_{j-1}$ 中找**最长相等的前后缀**
    - 从长到短考察每个前缀，看能否匹配后缀
    - 时间复杂度：`O(m^2)`
  - 高效方法：**递推**计算 $next$ 函数
    - 令 $next(0)=-1$
    - 假定已知 $next(j)=k$ , 即 $p_0...p_{j-1}$ 的最长相等前后缀为 $p_0...p_{k-1} = p_{j-k}...p_{j-1}$ ,其长度为 $k$
    - 现则要求 $next(j+1)$ , 即 $p_0...p_j$ 的最长相等前后缀的长度
      - 如果 $p_k=p_j$ ：
        - $p_0...p_j$ 的最长相等前后缀为 $p_0...p_k = p_{j-k}...p_j$
        - 其长度 $next(j+1)=k+1$
      - 如果 $p_k \neq p_j$ ：考察 $p_0...p_{j-1}$ 中**稍短**一点（**第2长**）的相等前后缀（假定其长度为 `𝒌′` ），然后比较这两个前后缀的下一个字符 $𝒑_{𝒌′}$ 和 $p_j$ ：
          - 若 $𝒑_{𝒌′} = p_j$ ：则 $next[j + 1] = 𝒌′+1$
          - 若 $𝒑_{𝒌′} \neq p_j$ ：考察 $p_0...p_{j-1}$ 中**更短**一点（**第3长**）的相等前后缀（假定其长度为 `𝒌′′` ），然后比较这两个前后缀的下一个字符 $𝒑_{𝒌′′}$ 和 $p_j$ ：
            - 若 $𝒑_{𝒌′′} = p_j$ ：则 $next[j + 1] = 𝒌′′+1$
            - 若 $𝒑_{𝒌′′} \neq p_j$ ：考察 $p_0...p_{j-1}$ 中**更短**一点（**第4长**）的相等前后缀 $......$
    - 故有：
      ```cpp {.line-numbers}
      while(p[k] != p[j])
        k = p[0]...p[j-1]中更短一点的相等前后缀的长度
      next[j + 1] = k + 1;
      ```
      - 而已知 $next(j)=k$ ，则 $p_0...p_{j-1}$ 中更短的相等前后缀：从 $p_0...p_{j-1}$ 第2长相等前后缀得出即求 $p_0...p_{k-1}$ 的最长相等前后缀，最后知即是 $next[k]$ ，故：
       ```cpp {.line-numbers}
       while(k >= 0 && p[k] != p[j])
            k = next[k];
       next[j + 1] = k + 1;
       ```
    - 故 $next$ 数组的计算的实现代码：
      ```cpp {.line-numbers}
      void buildNext(char *P, int next[], int m){
        int k = next[0] = -1;
        for(int j = 0; j < m - 1; j++){ // 求next[j+1], j+1<m
            while(k >= 0 && P[k] != P[j]) // 关键运算字符比较 执行次数取决于 k=next[k] 和 next[j+1]=++k 的次数
                k = next[k]; //求 p[0]…p[k-1] 的最长相等前后缀(使 k 减小)
            next[j + 1] = ++k; // (使 k 增加)
        }
      }
      ```
      - 时间复杂度分析:
        - 字符比较次数取决于 **k 增加次数**和 **k 减小次数**之和
        - `++k` 在 `for` 循环内, 执行次数不会超过 `m` 次, 即 `k` 增加不超过 `m` 次, 每次加 1
        - `k` 初值 `-1` , 迭代过程中永远不会小于`-1` , 故 **`k` 减小（ `k =next [k]` ）的次数不会超过 `k` 增加的次数**（即 `m` 次）, 否则 `k` 就比 -1 小了
        - 故关键运算不会超过 `2m` 次, 即 `O(m)`
      - 总结：字符集越小，字符单次成功比对概率 $Pr$ 越高 ，KMP 算法相对于朴素匹配算法优势越大（ $Pr = 1 / k$ ，$k$ 为字符集大小）

## 拓展
- 问题：给定目标串 $S$ 和模式串 $P$ ，输出 $P$ 在 $S$ 中出现的次数，以及 $P$ 在 $S$ 中的所有匹配位置
- 示例代码：
```cpp {.line-numbers}
int KMP(char S[], char P[], int n, int m){ // S 长度 n ，P 长度 m
    int next[N], i = 0, j = 0, cnt = 0; // cnt 为 P 在 S 中出现次数
    buildNext(P, next, m); //计算 next 数组, 需要多算一位到 next[m]
    while(j < m && i < n){
        if(j == -1 || S[i] == P[j]) i++, j++;
        else j = next[j]; // 确定 P 下次匹配位置
        if(j == m){ // 匹配成功
            cnt++; printf("%d\n", i - m); // 输出此时 P 在 S 中位置
            j = next[j]; // “假装” 此处失配，重新确定下次匹配位置
        }
        return cnt;
    }
}
```

### KMP算法的隐藏应用举例
- 引入分析：（字符串长度为 $n$ ）
  - 最长相等的前后缀长度：$next[n]$
  - 第二长相等的前后缀长度：$next[next[n]]$
  - 最长重复前缀：$next$ 数组中的最大值 $max_i(next[i])$
- **回文前缀**：
  - **问题 1**：给定长度为 `n` 的字符串 `p` ，编写算法返回 `p` 的最长回文前缀的长度，如果 `p` 本身就是回文，则返回 `p` 的长度 (假定 `p` 的最长回文前缀为 $p_0...p_k$ ，则有 $p_0...p_k = p_k...p_0$ )
  - **问题 2**：给定长度为 `n` 的字符串 `p` ，在 `p` 前面最少添加几个字符可使 `p` 变为回文串，并输出该最短回文串
    - 参考题目：[LeetCode 214 最短回文串](https://leetcode.cn/problems/shortest-palindrome/description/)
- **循环节**：
  - 引入：如果一个字符串 `S` 可由它的一个最小子串 `P` 重复 `k` 次构成，则 `P` 称为 `S` 的循环节，`k` 称为循环周期。例如：`S= a b c a b c a b c a b c` 的循环节（最小重复单元）为 `a b c` ，循环周期为 `4`
  - 问题：给定一个非空字符串 `S` ，检查它可否可以通过由它的一个子串重复多次构成
  - 参考题目：[LeetCode 459 重复的子字符串](https://leetcode.cn/problems/repeated-substring-pattern/description/)
  - 算法实现：
    - 如果 $next[n]>0$ 且 `n%L==0` ，则 $S$ 可由循环节完全循环构成
    - 循环节长度为：$L=n-next[n]$ ，循环周期为： $n/L$
    - 示例代码：
      ```cpp {.line-numbers}
      const int N = 1e4 + 10;
      bool repeatedSubstringPattern(char *s){
        int next[N], n = strlen(s);
        buildNext(s, next, n);
        int L = n - next[n];
        if(next[n] > 0 && n % L == 0) return true;
        return false;
      }
      ```

## KMP算法的改进
- 原始 $next$ 函数的不足
  - 如果在 $p_j$ 处失配，必有 $s_i \neq p_j$
  - 如果 $p_j = p_k$ ，则必有 $s_i \neq p_k$ ，下趟匹配必然失败
  - 改进方法：在确定下次匹配位置时，除了找失配位置前的最长相等前后缀之外，还应比较其后面的两个字符（即前缀后面的字符 $p_k$ 和当前失配位置的字符 $p_j$ ），确保 $p_j \neq p_k$
- 改进的 $next(j)$ 函数
  - 额外要求：最长相等前后缀后面的两个字符（即前缀后面的字符 $p_k$ 和当前位置 $j$ 的字符 $p_j$ ）不能相等，即 $p_j \neq p_k$
  - 则有改进函数：
   $$
    nextval(j) = 
    \begin{cases}
    -1, & j = 0 \\
    \max\left\{ k \mid p_0 \dots p_{k-1} = p_{j-k} \dots p_{j-1} \text{ 且 } p_k \neq p_j, 0 < k < j \right\}, & \text{存在}k \\
    0, & p_0 \dots p_{j-1} \text{ 不存在相等的前后缀但 } p_0 \neq p_j \\
    -1, & p_0 \dots p_{j-1} \text{ 不存在相等的前后缀且 } p_0 = p_j
    \end{cases}
   $$
    - $nextval(j) = 0$ 时，说明至少可以将 $P_0$ 移动到失配位置，与 $S$ 进行下一次比对
    - $nextval(j) = -1$ 时，说明 $P_j$ 前所有位置都不可能与 $S$ 匹配 ， 应将 $P$ 整体移过失配位置
  - 启示：更加充分地利用以往比对所提供的信息
    - $next$ ：以往成功比对所获得的“经验”——$𝑺_{𝒊−𝒋}… 𝑺_{𝒊−𝟏}$是什么
    - $nextval$ ：以往失败比对所吸取的“教训”——$𝑺_𝒊$ 不是什么
- $nextval$ 函数的实现
  ```cpp {.line-numbers}
    void buildNextval(char P[], int nextval[], int m){
        int k = nextval[0] = -1;
        for(int j = 0; j < m; j++){ //求 nextval[j+1]
            while(k >= 0 && P[k] != P[j])
                k = nextval[k];
            if(P[j + 1] != P[k + 1]) nextval[j + 1] = ++k;
            else nextval[j + 1] = nextval[++k];
        }
    }
  ```
